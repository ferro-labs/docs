title: Provider configuration
description: Environment variables for each provider.
---

The gateway registers providers based on environment variables. Set at least one provider key before starting the server.

## Core provider keys

- `OPENAI_API_KEY`
- `ANTHROPIC_API_KEY`
- `GROQ_API_KEY`
- `GEMINI_API_KEY`
- `MISTRAL_API_KEY`
- `TOGETHER_API_KEY`
- `COHERE_API_KEY`
- `DEEPSEEK_API_KEY`
- `PERPLEXITY_API_KEY`
- `FIREWORKS_API_KEY`
- `AI21_API_KEY`

## Azure OpenAI

Azure requires additional fields:

```bash
export AZURE_OPENAI_API_KEY=...
export AZURE_OPENAI_ENDPOINT=https://your-resource.openai.azure.com
export AZURE_OPENAI_DEPLOYMENT=your-deployment
export AZURE_OPENAI_API_VERSION=2024-10-21
```

## Ollama

Ollama does not need an API key.

```bash
export OLLAMA_HOST=http://localhost:11434
export OLLAMA_MODELS=llama3.2,llama3.1
```

## Replicate

```bash
export REPLICATE_API_TOKEN=...
export REPLICATE_TEXT_MODELS=meta/llama-3-8b-instruct
export REPLICATE_IMAGE_MODELS=stability-ai/sdxl
```

## AWS Bedrock

Bedrock uses the AWS credential chain. Set `AWS_REGION` or AWS credentials:

```bash
export AWS_REGION=us-east-1
export AWS_ACCESS_KEY_ID=...
export AWS_SECRET_ACCESS_KEY=...
```

## Validate providers

Check `/v1/models` for the current model list and `/health` for provider availability.
