title: Configuration
description: Configure routing, targets, plugins, and aliases.
---

The gateway loads configuration from a JSON or YAML file referenced by `GATEWAY_CONFIG`.

## Load a config file

```bash
export GATEWAY_CONFIG=./config.yaml
```

Supported extensions are `.json`, `.yaml`, and `.yml`.

## Core fields

```yaml
strategy:
  mode: fallback  # single | fallback | loadbalance | conditional

targets:
  - virtual_key: openai
    retry:
      attempts: 3
  - virtual_key: anthropic

aliases:
  fast: gpt-4o-mini
  smart: claude-3-5-sonnet-20241022

plugins:
  - name: word-filter
    type: guardrail
    stage: before_request
    enabled: true
    config:
      blocked_words: ["password", "secret"]
      case_sensitive: false
```

## Strategy modes

- `single`: always route to the first target
- `fallback`: try targets in order, retrying on failure
- `loadbalance`: distribute by weight across targets
- `conditional`: route based on request conditions

### Conditional rules

Each rule uses a request field and value to pick a target:

```yaml
strategy:
  mode: conditional
  conditions:
    - key: model
      value: gpt-4o-mini
      target_key: openai
    - key: model
      value: claude-3-5-sonnet-20241022
      target_key: anthropic
```

## Targets

Targets reference provider keys registered by environment variables. Use `weight` for load balancing and `retry` or `circuit_breaker` for resiliency.

```yaml
targets:
  - virtual_key: openai
    weight: 70
    retry:
      attempts: 2
    circuit_breaker:
      failure_threshold: 5
      success_threshold: 1
      timeout: "30s"
```

## Aliases

Aliases map friendly names to concrete model IDs. Aliases cannot point to other aliases.
